注册流程

  1. 用户输入账号密码，校验码（todo）

  2. 校验用户的账户、密码、校验密码是否合规

    1. 账户不小于6位
    2. 密码不小于8位
    3. 账户不能重复
    4. 账户不包含特殊字符
    5. 密码和校验密码相同

  3. 对密码进行加密

  4. 向数据库插入用户数据

    工具包：apache.common.lang3， 提供了很多重用的公共方法

流程中注意对性能的处理，比如账户不能重复中会进行查库的操作可以放到最后进行

登录流程

接受参数：用户账户，密码

请求类型：POST

返回值：用户信息（脱敏）

逻辑

  1. 校验用户账户和密码是否合法

  2. 校验密码是否输入正确（之后可以做限流（看一下最近这个账户登录过多少次，如果登录次数太多，限制一下））

    逻辑删除

  3. 记录用户的登录态（session），存储到服务器上（用SpringBoot框架封装的服务器tomcat记录）

  4. 返回用户信息（脱敏）

如何知道是哪个用户登录 了

（javaweb）

  1. 链接服务器端后，获得一个session状态返回给前端
  2. 登录成功后，得到登录成功的session，给session设置一些值（比如用户信息），返回给前端一个设置cookie的命令
  3. 前端设置cookie到浏览器
  4. 前端再次请求后端的时候，再请求头中带上cookie去请求
  5. 从session中取出基于该session存储的变量
