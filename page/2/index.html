Java 8 Collection.removeIf()

以前都是用 迭代器实现集合中删除的操作，发现一个方法叫 removeif 底层使用的迭代器实现了删除操作，于是创建测试一下这个方法

创建了一个list 使用 Arrays.asList() (然后就在这上面踩坑了)

然后使用 removeif 方法

抛出了 UnsupportedOperationException, 震惊 再次查看源码，感觉逻辑上没有问题。

在 iterator 中的 remove() 时会出现问题，跟随断点进去发现直接调用到了父类 abstractList 的 remove() 方法。

到这里 大概就明白了问题，其实是 asList 方法创建的 java.util.Arrays.ArrayList 没有实现 remove 方法，与 java.util.ArrayList 不同。我们使用 new ArrayList(Arrays.asList()) 解决问题。

removeIf 比起自己写迭代器删除还是方便了很多的。

Java 语言中的类、方法、变量、参数和包等都可以被注解修饰。

Annotation

注解是一个接口 有三个主要部分

  1. Annotation

    每 1 个 Annotation 对象，都会有唯一的 RetentionPolicy 属性；至于 ElementType 属性，则有 1~n 个。

  2. ElementType 是 Enum 枚举类型，它用来指定 Annotation 的类型。

指定一个 Annotation 对象使用在什么类型上

  1. RetentionPolicy

    用来指定 Annotation 的作用域。

Annotation 通用定义

@interface

使用 @interface 定义类时，意味着这个类实现了 java.lang.annotation.Annotation 接口，表示这个类是一个Annotation。定义 Annotation 时，@interface 是必须的。注解不能继承其他的注解或接口。

@Retention

指定 Annotation 的策略属性，是只在代码中，还是编入class文件中，或者是在运行时可以通过反射访问，其中属性就是 RetentionPolicy 的类型。

@Documented

标记这些注解是否可以出现在 javadoc 中。

@Target

指定 Annotation 的类型属性，类型是 ElementType 。

@Inherited - 标记这个注解是继承于哪个注解类(默认注解并没有继承于任何子类)
