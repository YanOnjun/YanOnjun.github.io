面向过程比面向对象性能高？

发表于 2023-05-29

Java 性能差的原因是因为 Java 是半编译语言，最终的执行代码不是可以直接被 CPU 执行的二进制编码。

面向过程语言大多数是直接编译成机械码在电脑上执行。而且面向过程性能也不一定比面向对象性能好。

设计范式和性能无关的。主要是编程语言的运行机制决定的。

接口和抽象类的区别

发表于 2023-05-29

相同点

  • 都不能被实例化只能通过 继承/实现 [[抽象类#^bdf134]][[接口#^e92cf9]]
  • 都可以有抽象方法

区别

  • 接口多继承，抽象类单继承
  • 接口的类、方法、成员属性都是 public 修饰，抽象类可以使用其他的修饰符
  • 接口中不能有静态代码块 （Java 8 可以有静态方法）

重载

发表于 2023-05-29

重载指的是在一个类里面，对方法的重载，方法名字相同，参数不同，返回类型可以相同可以不同。但是无法使用返回类型作为重载函数的标准。

重写

发表于 2023-05-29

重写是对父类允许访问的方法的内容的重新实现。

规则

  • 能重写的先决条件是继承关系
  • 参数列表必须完全相同
  • 返回类型必须是重写方法返回值的派生类
  • 访问权限不可以低于重写的方法
  • 只能重写父类的方法
  • 声明为 final 的方法不能重写
  • 声明为 static 的方法不能重写
  • 子类只能重写父类可以访问的方法 （访问修饰符的级别）
  • 重写的方法可以抛出非强制性异常，不可以抛出比被重写方法声明更广泛的强制性异常
  • 不能重写构造方法，具体可以查看[[继承#构造器]]

继承

发表于 2023-05-29

不同类型的对象，相互之间有一些共同点。同时每一个对象又定义了额外的特性使他们与众不同。继承是使用父类的定义作为基础建立子类的思想，子类拥有父类的特征和行为，子类可以增加新的数据或者新的功能，但是不能选择性的继承父类，只能全部继承。

继承符合的关系是 is-a

使用继承可以快速创建新的类，提高代码重用性，程序的可维护性。

特性：

  1. 子类拥有父类所有的属性和方法，但是不能访问私有属性和方法。
  2. 子类可以拥有自己的属性和方法。
  3. 子类可以重新实现父类的方法。
  4. 所有类默认继承 Object（Java)
  5. Java 的继承是单继承，但是可以多重继承

多重继承

多重继承指的是一个父类还有一个父类，子类也可以继承到父类的父类的属性和方法（C++中没有这个特性）

实现

实现是继承的一种，是类与[[接口]]之间的继承关系，由于接口是一组方法的集合，所以用实现方法而不是继承方法更符合描述。

构造器

Java: 子类不继承父类的构造器，只是在实例化时显式或隐式调用。父类如果有有参构造器，那么子类的构造器中必须显式的使用 super 方式调用，如果是无参构造器（Java默认没有写构造器的话也有一个无参构造器），则不需要调用，系统会自动调用父类的无参构造器。

接口

发表于 2023-05-29

接口是一种特殊的[[抽象类]]，是只包含抽象方法的集合，一个类通过继承接口的方式，继承实现接口的抽象方法。 ^e92cf9

接口并不是类，编写接口的方式和类很相似，但是不是同一个概念，接口只包含类要实现的方法。除非实现接口的类是抽象类，否则这个类要定义接口的所有方法。

特性：

  • 接口是隐式抽象的，当声明一个接口的时候，不必使用 abstract 关键字
  • 接口中不能有静态代码块
  • 接口中的方法隐式指定为 public abstract （其他访问修饰符会报错）
  • 接口中的变量隐式指定为 public static final （同上）

接口的方法

Java 8 之后 Interface 可以用 default 和 static 修饰方法，这样的方法可以有方法体，实现类也不用重写方法。

  1. default 修饰的方法可以通过 this 调用，也可以被[[继承]]、[[重写]]。
  2. static 修饰的方法与静态方法一样，但是不能被[[继承]]。

接口的实现

  • 一个非抽象类实现接口的同时，必须同时实现接口中的方法。
  • 一个类可以实现多个接口。
  • 一个接口可以继承多个接口，类的多继承不合法，接口可以实现多继承，子接口继承父接口的方法。

如果一个类实现了两个接口，并且这两个接口有同一个 default 方法，那么这个类就必须重写这个方法。否则编译会报错。

抽象类

发表于 2023-05-29

有一些类不是用来描绘对象的，这样的类没有足够的信息，就是抽象类。

特点

  • 抽象类不能进行实例化，但是其他类的功能依然存在：成员变量、成员方法、构造方法… ^bdf134
  • 由于抽象类不能实例化，所以必须继承才能使用抽象类。
  • 抽象类表示一种继承关系，一个类只能继承一个抽象类。

抽象方法

抽象方法只能存在于抽象类里，抽象方法不用实现（使用大括号的具体方法实现）。

封装

发表于 2023-05-29

封装是把一个对象的属性隐藏在对象内部，不允许外部直接访问对象的内部信息，通过一些可以被外界访问的方法来操作属性。

多态

发表于 2023-05-29

多态是一个对象有多种状态，具体表现为父类的引用指向子类的实例。

多态的特点：

  • 对象类型和引用类型之间具有 [[继承]]/[[继承#实现]] 的关系
  • 引用类型变量的方法调用的是哪个类里的方法，在程序运行期间才能确定
  • 引用类型变量不能调用子类扩展的方法
  • 如果子类[[重写]]了父类的方法，调用的子类重写的方法，如果没有重写，调用的是父类的方法

注意（个人观点）

[[重载]] 不是多态。
从运行时期来说：多态的深层含义是运行时调用的是父类还是子类，重载是编译期时对不同参数表的方法的修饰形成不同的方法，对于这些不同的方法，再编译期间就决定了调用哪一个，被称为静态绑定。
从对象来说：广泛上的多态指的是类之间因为继承/实现的先决条件产生的，而重载针对的方法，可以在父子类中重载，也可以不用继承重载自身的方法。

如果它不是晚绑定，它就不是多态。
——《Java 编程思想》

在面向对象编程中，方法的绑定可以在编译时或运行时进行。编译时绑定也称为早绑定（静态绑定），运行时绑定也称为晚绑定（动态绑定）。
