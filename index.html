String

发表于2023-06-16更新于2023-06-16

String 是 Redis 中最简单最常用的一个数据结构。

用来存储任何类型的数据：字符串、整数、浮点数、图片（base64编码或者路径）、序列化后的对象。

Redis中的字符串使用的不是 c 语言原生的字符串，而是自己实现的一种数据结构：简单动态字符串（SDS）。相比于原生字符串，SDS 不仅可以保存文本数据，还可以保存二进制数据，获取字符串长度的复杂度为O(1)，并且 API 安全，不会造成缓冲区溢出

常用命令

命令 介绍
SET key value 设置指定 key 的值
SETNX key value 只有在 key 不存在时设置 key 的值
GET key 获取指定 key 的值
MSET key1 value1 key2 value2 … 设置一个或多个指定 key 的值
MGET key1 key2 … 获取一个或多个指定 key 的值
STRLEN key 返回 key 所储存的字符串值的长度
INCR key 将 key 中储存的数字值增一
DECR key 将 key 中储存的数字值减一
EXISTS key 判断指定 key 是否存在
DEL key（通用） 删除指定的 key
EXPIRE key seconds（通用） 给指定 key 设置过期时间

应用场景

  1. 存储常规数据
    缓存 session、token、图片地址、序列化后的对象。

  2. 计数
    用户单位时间的请求数（简单限流可以用到）、页面单位时间的访问数。

  3. 分布式锁
    利用 SETNX key value 命令可以实现一个最简易的分布式锁。

Redis 持久化

发表于2023-06-16更新于2023-06-16

缓存的数据存储在内存中，但是当机器关机、宕机之后，数据会丢失，Redis 于是实现了数据的持久化，将数据写入到硬盘之中。

Redis 提供了三种持久化方式

  • 快照 RDB
  • 追加文件 AOF
  • RDB 和 AOF 的混合持久化

快照持久化是 Redis 默认采用的持久化方式，在 redis.conf 文件中有配置

RDB

Redis 通过两个命令来生成 RDB 快照

  • save：同步保存，会阻塞主进程
  • bgsave：通过子进程执行保存，不会阻塞

Redis 基础

发表于2023-06-16更新于2023-06-16

Redis介绍

Redis 是基于 C 语言开发的 NoSQL 数据库，存储 KV 键值对数据，数据存储在内存中，读写速度非常快。

个人学习可以通过本机安装或者使用 try redis 进行体验

数据结构：

  • 基础数据结构：[[String]]、List、Hash、Set、ZSet

SQL 和 NOSQL

SQL NoSQL
结构化 非结构化
关联 非关联（信息之间没有关联）
SQL查询 非SQL（没有语法）

参考链接

  • SQL 与 NoSQL还在傻傻分不清？

Redis 特点

  • 键值对类型存储
  • 基于内存
  • 有过期策略
  • 单线程，命令具有原子性
  • 支持[[Redis 持久化|持久化]]
  • 支持分布式（主从集群）

Redis 快的原因

  • Redis 基于内存（内存的访问速度是磁盘的上千倍）【主要原因】
  • Redis 基于 Reactor 模式设计实现的模式，主要单线程事件循环和 IO 多路复用
  • Redis 优化后的数据结构，性能很高

为什么用 Redis

  1. 高性能
    数据从数据库中获取的话，是从磁盘中获取，过程比较慢，使用 Redis 的话，可以直接从缓存中获取，直接操作内存，速度快
  2. 高并发
    Redis 的 QPS 很容易达到 10w+ ，Redis 集群会更高，而数据库的 QPS 在 1w左右，操作缓存能承受的能力是远大于数据库。也就提高了系统的并发。

    QPS( Query Per Second) : 每秒查询次数

Redis 应用

除了做缓存，Redis 还可以做：

  • 分布式锁
  • 限流
  • 消息队列 (不推荐 最好还是使用专门的消息队列)
  • …

面向过程比面向对象性能高？

发表于2023-05-29更新于2023-05-29

Java 性能差的原因是因为 Java 是半编译语言，最终的执行代码不是可以直接被 CPU 执行的二进制编码。

面向过程语言大多数是直接编译成机械码在电脑上执行。而且面向过程性能也不一定比面向对象性能好。

设计范式和性能无关的。主要是编程语言的运行机制决定的。

接口和抽象类的区别

发表于2023-05-29更新于2023-05-29

相同点

  • 都不能被实例化只能通过 继承/实现 [[抽象类#^bdf134]][[接口#^e92cf9]]
  • 都可以有抽象方法

区别

  • 接口多继承，抽象类单继承
  • 接口的类、方法、成员属性都是 public 修饰，抽象类可以使用其他的修饰符
  • 接口中不能有静态代码块 （Java 8 可以有静态方法）

重载

发表于2023-05-29更新于2023-05-29

重载指的是在一个类里面，对方法的重载，方法名字相同，参数不同，返回类型可以相同可以不同。但是无法使用返回类型作为重载函数的标准。

重写

发表于2023-05-29更新于2023-05-29

重写是对父类允许访问的方法的内容的重新实现。

规则

  • 能重写的先决条件是继承关系
  • 参数列表必须完全相同
  • 返回类型必须是重写方法返回值的派生类
  • 访问权限不可以低于重写的方法
  • 只能重写父类的方法
  • 声明为 final 的方法不能重写
  • 声明为 static 的方法不能重写
  • 子类只能重写父类可以访问的方法 （访问修饰符的级别）
  • 重写的方法可以抛出非强制性异常，不可以抛出比被重写方法声明更广泛的强制性异常
  • 不能重写构造方法，具体可以查看[[继承#构造器]]
